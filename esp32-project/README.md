# ESP32 Project Structure

This directory contains ESP-IDF project configuration for building EthervoxAI on ESP32 microcontrollers.

## Architecture: Symlinked Source

**Important:** The `src/` and `include/` directories in this folder are **symlinks** to the parent repository:

```bash
esp32-project/src -> ../src
esp32-project/include -> ../include
```

This means:
- ✅ Any changes to files in `../src/` or `../include/` **automatically** appear here
- ✅ Single source of truth - no duplication or manual syncing needed
- ✅ AI assistants and build tools can edit files in either location
- ❌ Do NOT copy files manually between `src/` and `esp32-project/src/`

## Setup

Run the setup script to create the ESP32 build environment:

```bash
./scripts/setup-esp32-toolchain.sh
```

This will:
1. Install ESP-IDF (v5.1) if not present
2. Create the symlinks (`src` → `../src`, `include` → `../include`)
3. Set up the ESP32 toolchain

## Building

```bash
# Source ESP-IDF environment (required in each new terminal)
. ~/esp/esp-idf/export.sh

# Configure target (ESP32-S3 recommended for AI workloads)
idf.py set-target esp32s3

# Configure project
idf.py menuconfig

# Build
idf.py build

# Flash and monitor
idf.py -p /dev/ttyUSB0 flash monitor
```

## Files in This Directory

- `CMakeLists.txt` - ESP-IDF project configuration
- `sdkconfig` - ESP32 build configuration (gitignored, generated by menuconfig)
- `build/` - Build artifacts (gitignored)
- `src/` - **Symlink to ../src** (shared source code)
- `include/` - **Symlink to ../include** (shared headers)
- `main/` - ESP32-specific entry point (if needed)

## For AI Coding Assistants

When working on EthervoxAI code:
- Edit files in `src/` or `include/` at the repository root
- Changes automatically apply to both desktop and ESP32 builds
- No special handling needed for `esp32-project/` directory
- .gitignore patterns apply to `src/` only (symlinks follow parent patterns)
